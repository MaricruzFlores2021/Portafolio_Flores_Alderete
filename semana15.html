Proyecto de investigación


La aplicación web fullstack propuesta se enfoca en dos módulos clave. Utilizando NEXT JS, se creará un catálogo de productos con diseño responsivo y paginación, incorporando elementos visuales como un header y un footer. En el backend, se implementará una API Java y Spring para suministrar datos JSON de productos y usuarios, junto con un módulo de administración que incluirá autenticación, dashboard y un CRUD restringido.


PROYECTO DE INVESTIGACIÓN



• FRONTED:
El trabajo consiste en la creación de una aplicación web fullstack con funcionalidades específicas. En el módulo híbrido del catálogo de productos, se empleará NEXT JS para desarrollar un catálogo de productos con diseño responsivo. Este catálogo exhibirá los primeros 20 productos con estilos de tailwind, organizados con paginación de 20 en 20. Cada producto estará representado en un componente card del lado del servidor, y al seleccionar un producto, se abrirá una página detallada con la imagen ampliada del mismo. Además, se incorporará un header que presentará el logo y el nombre de la empresa, así como un footer con paginación automática actualizada mediante eventos de desplazamiento del mouse.

• BACKEND:
En el módulo backend de productos, se implementará una API utilizando Java y Spring, aprovechando Spring Data y MySQL para suministrar documentos JSON desde las tablas de productos y usuarios. También, se desarrollará un módulo de administración con un dashboard que contendrá opciones para gestionar usuarios y productos. La autenticación será ejecutada mediante Next JS, interactuando con la API previamente mencionada. Adicionalmente, se diseñará un CRUD restringido con un middleware para administrar la tabla de productos, brindando así una solución completa y robusta para la aplicación web propuesta.

• PROCEDIMIENTO: El procedimiento para desarrollar la aplicación web fullstack comienza con la utilización de NEXT JS para crear un catálogo de productos responsivo y estilizado con tailwind, organizado con paginación y detallado en componentes card del lado del servidor y cliente. El backend se implementa con Java y Spring, proporcionando una API para gestionar productos y usuarios, y un módulo de administración con autenticación mediante Next JS. Se incorpora un dashboard con opciones para usuarios y productos, y un CRUD restringido para la tabla de productos, garantizando la seguridad y eficiencia en la manipulación de la información. Este enfoque integral asegura la creación de una aplicación completa y robusta.

• RESULTADOS: Los resultados de este proceso serán una aplicación web fullstack completamente funcional con un catálogo de productos responsivo y detallado, organizado con paginación, estilos atractivos y componentes tanto del lado del servidor como del cliente. Además, se implementará un backend robusto utilizando Java, Spring y MySQL, con una API para gestionar productos y usuarios. El módulo de administración contará con un dashboard, opciones de autenticación y un CRUD restringido para la gestión eficiente de productos. En resumen, se obtendrá una aplicación web integral con características avanzadas tanto en el frontend como en el backend.
