HyperText Markup Language (HTML)


Se brinda una completa perspectiva sobre HTML, el lenguaje de marcado fundamental para la estructura y semántica de las páginas web. Desde su definición hasta su funcionamiento, se exploran las etiquetas clave y DOM.

HYPERTEXT MARKUP LANGUAGE (HTML)



• DEFINICIÓN: Es un lenguaje de marcado que define la estructura y semántica de las páginas web mediante etiquetas como "<>". Almacenado en archivos .html o .htm, sigue el estándar SGML y utiliza el Modelo de Objeto de Documento (DOM) para organizar la estructura de la página.

• FUNCIONAMIENTO: Las páginas web se construyen con elementos que tienen etiquetas y atributos. Las etiquetas indican el inicio y fin de cada componente, mientras que los atributos describen sus características.

• PRINCIPALES ETIQUETAS: Utiliza etiquetas como 'doctype html', 'html', 'head', y 'title' para definir la estructura y metadatos. Además, emplea 'base' para establecer la URL base, 'link' para enlazar hojas de estilo CSS externas, y 'meta' para definir metadatos. La etiqueta 'style' permite escribir CSS directamente en línea. Además, 'body' para el contenido principal, 'section' y 'nav' para secciones y navegación, 'article' para contenido autónomo, 'aside' para información relacionada, y 'h1' a 'h6' para títulos. 'header' y 'footer' son encabezados y pies de página, 'address' contiene información de contacto, y 'main' indica el contenido principal.

• XML O LENGUAJE DE MARCADO EXTENSIBLE: Es un estándar para almacenar y compartir información estructurada entre plataformas. Define la estructura y el significado de los datos en documentos divididos en prolog (metadatos) y body (partes estructural y de contenido).

• DOM: Es una interfaz para documentos HTML y XML que se representa como un árbol, permitiendo la manipulación con lenguajes de scripting como JavaScript. En el proceso de renderización de un documento HTML, se utiliza el Critical Rendering Path, que inicia con la creación del render tree, combinando el DOM (Modelo de Objetos del Documento) y el CSSOM (Modelo de Objetos de Estilo CSS). El render tree excluye elementos no visibles, como los con estilo display: none, y se utiliza como base para el renderizado final en el navegador. Por otro lado, el DOM tiene como objetivo convertir la estructura y contenido del HTML en un árbol de nodos, donde la etiqueta html actúa como la raíz y los elementos anidados como ramas, proporcionando un modelo de objeto utilizado por diversos programas.

• CICLO DE VIDA DE UNA PÁGINA: La ruta de renderizado crítica, también conocida como Critical Rendering Path, es la secuencia de pasos que sigue el navegador para convertir HTML, CSS y JavaScript en píxeles en la pantalla. Comienza con la solicitud de una página web, donde se crea el Document Object Model (DOM) a partir de la respuesta HTML, se genera el CSS Object Model (CSSOM), y ambos se combinan en el árbol de renderizado (Render Tree) para obtener los estilos del contenido visible. Luego, se realiza el cálculo del diseño y, finalmente, se lleva a cabo la representación o "pintado" en la pantalla. Este proceso es crucial para la visualización eficiente de una página web.
