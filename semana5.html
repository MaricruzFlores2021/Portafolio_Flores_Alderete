Tema de la Semana 5 : Desarrollo con Framework JS y Estilos


En esta parte se aborda el desarrollo FrontEnd con React JS, destacando conceptos clave como Cliente-Side Rendering y técnicas de estilización con Styles Modules y Styled Components.


DESARROLLO FRONTEND CON FRAMEWORK JS Y ESTILOS



• DESARROLLO DE APLICACIÓN FRONTEND CON FRAMEWORK:
o FRAMEWORKS POPULARES: Los frameworks JS populares incluyen React, Angular y Vue.js, cada uno con sus propias características y enfoques.
o REACT: React es conocido por su virtual DOM y su enfoque en componentes reutilizables. A continuación, se muestra más información:
 DEFINICIÓN: React JS es una potente biblioteca de JavaScript para crear interfaces interactivas en aplicaciones de una sola página. Se destaca por su enfoque en la reutilización de componentes, mejorando la mantenibilidad del código y permitiendo la construcción eficaz de aplicaciones complejas.
 FUNCIONAMIENTO: Funciona eficientemente con un DOM virtual en memoria, minimizando cambios en el DOM del navegador para una actualización precisa y eficiente de la interfaz de usuario.
 HERRAMIENTAS: El desarrollo con React implica herramientas clave como Babel, ESLint, Sass/Scss, Font Awesome y Webpack, formando un ecosistema integral. Para simplificar la instalación, se utilizan herramientas como Create React App, Vite y Next.js, facilitando el desarrollo y despliegue de aplicaciones React.

• ESTRUCTURA Y OPERACIÓN DE FRAMEWORK - CLIENT SIDE RENDERING (CSR):
o En el CSR, el navegador descarga el código JavaScript y es el cliente quien se encarga de la representación de la interfaz. Esto permite actualizaciones rápidas y reactivas en la interfaz de usuario sin tener que recargar toda la página.

• GESTIÓN DE DEPENDENCIAS / HERRAMIENTAS DE CREACIÓN DE PROYECTOS:
o GESTIONAR DEPENDENCIAS: npm (Node Package Manager) o Yarn se utilizan para gestionar dependencias y scripts de construcción.
o CLI: Las CLI de los frameworks (por ejemplo, Create React App para React) facilitan la creación y configuración inicial de proyectos.

• DESARROLLO BASADO EN COMPONENTES - JSX / PROPS / CHILDREN:
o JSX: Es una extensión de JavaScript que permite escribir HTML de manera similar a XML, facilitando la construcción de componentes de manera declarativa.
o PROPS (propiedades): Permiten la transferencia de datos de un componente padre a un componente hijo.
o CHILDREN: Se utiliza para pasar elementos o componentes hijos a otros componentes.

• UTILIZACIÓN DE ESTILOS EN FRAMEWORK JS:
o Styles Inline: Estilos directamente en el atributo style de las etiquetas. Aunque puede ser útil para pequeñas aplicaciones, puede volverse complicado en proyectos más grandes.
o Style Sheets: Uso de archivos CSS externos para estilos globales y específicos de la aplicación.
o Styles Modules: Módulos de estilos locales para evitar conflictos de nombres de clase en componentes.
o Styles Components: Librerías como Styled Components permiten definir estilos junto con componentes, proporcionando encapsulamiento de estilos.

• PROCEDIMIENTO: Se proporciona una guía inicial para trabajar con React, comenzando con el uso de enlaces CDN para obtener las librerías React y ReactDOM. Luego, se muestra cómo integrar React directamente en HTML para propósitos de desarrollo, aunque se destaca que para producción se requiere npm y Node.js. Se procede a la instalación de Node.js a través de un asistente de instalación paso a paso. La creación de un proyecto en React se presenta a través de tres opciones: Create React App, Vite y Next.js. Se exploran las estructuras de proyecto para cada opción, y se sugieren extensiones para Visual Studio Code y Google Chrome que facilitan el desarrollo y la depuración.

• RESULTADOS:Obtención de conocimientos del funcionamiento de React, la instalación de Node.js, la instalación React mediante las herramientas Vite, Remix, NextJS y la creación React App, y la comparación de las estructuras de directorio.
