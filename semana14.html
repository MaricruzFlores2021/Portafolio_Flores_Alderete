Tema de la Semana 14 :
Registro de microservicios en eureka y consumo


Eureka simplifica el registro y descubrimiento de microservicios, proporcionando un servidor central para el registro automático. Esto permite una expansión dinámica y una comunicación eficiente entre servicios, con opciones de configuración avanzada y adaptabilidad a entornos en la nube. Eureka se convierte en un componente clave para la gestión dinámica de servicios en infraestructuras basadas en la nube.


REGISTRO DE MICROSERVICIOS EN EUREKA



• Registro de Microservicios en Eureka: Detalles Expandidos
o Creación del servidor Eureka:
 Medios: Utilizando Spring Cloud, se establece un servidor Eureka como punto central para el registro y descubrimiento de microservicios.
 Configuración: La configuración implica definir el servidor en la aplicación principal y anotar la clase con @EnableEurekaServer.
 Propiedades: Se establecen propiedades como puerto y ubicación para garantizar la accesibilidad y la identificación única del servidor.
o Registro de un microservicio:
 Usos: Cada microservicio utiliza la anotación @EnableDiscoveryClient para registrarse automáticamente en Eureka.
 Especificaciones: Se especifica el nombre, instancia, dirección y metadatos del servicio para facilitar su identificación y conexión.
 Monitoreo: La salud del servicio se monitorea mediante rutas específicas, permitiendo una gestión efectiva de la disponibilidad.
o Resumen del registro de microservicio:
 Importancia: El registro centralizado permite la fácil expansión y contracción de servicios en un entorno distribuido.
 Ventajas: Eureka proporciona un panel de control para supervisar el estado y las instancias de los microservicios registrados.
 Impacto: La información de registro facilita la resiliencia, ya que otros servicios pueden adaptarse a los cambios dinámicos.
o Consumo de un Microservicio:
 Proceso: Los microservicios consultan el servidor Eureka para descubrir y consumir otros servicios disponibles.
 Medios: La carga equilibrada se logra automáticamente, distribuyendo las solicitudes entre las instancias registradas.
 Importancia: La comunicación es más flexible y eficiente, ya que los servicios se actualizan dinámicamente según su disponibilidad.
o Configuración avanzada en Eureka:
 Ajustes: Ajustes de tiempo de espera y renovación garantizan la precisión y la actualización oportuna de los registros.
 Importancia: Las estrategias de renovación evitan la expiración prematura del registro, asegurando la estabilidad del sistema.
 Impacto: La configuración de parámetros de salud permite una supervisión efectiva y la identificación de servicios en problemas.
o Configuración en la nube:
 Entorno: Para entornos en la nube, se integra Eureka con servicios de registro específicos de plataformas como AWS Service Discovery.
 Adaptación: La adaptación de la configuración implica consideraciones sobre seguridad, escalabilidad y redundancia en entornos en la nube.
 Importancia: Eureka se convierte en un componente clave para gestionar dinámicamente servicios en infraestructuras basadas en la nube, proporcionando una capa adicional de abstracción.

• PROCEDIMIENTOS: En este procedimiento, se inicia el servidor de la API mediante el comando .\mvnw spring-boot:run y se realiza la configuración del entorno, incluyendo la conexión a la base de datos MySQL. La implementación del modelo CursoModel define la estructura de la entidad "Curso," mientras que la interfaz CursoRepository y la clase CursoService manejan las operaciones CRUD de la base de datos. El controlador CursoController expone endpoints REST para la manipulación de cursos, permitiendo la recuperación, almacenamiento, visualización y eliminación de datos. Este conjunto de archivos y configuraciones conforma una API funcional, listo para interactuar con aplicaciones front-end u otros sistemas consumidores.

• RESULTADOS: La sesión ha producido avances significativos en el desarrollo del backend del sitio web, implementando una API REST funcional con Spring. Se cubrieron aspectos clave como la creación del servidor Eureka, el registro de microservicios, la configuración avanzada y en la nube. Además, se detallaron configuraciones específicas, como la conexión a la base de datos y operaciones CRUD mediante modelos, repositorios, servicios y controladores.
