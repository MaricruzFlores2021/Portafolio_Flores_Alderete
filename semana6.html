Tema de la Semana 6 : Desarrollo con Frameworks JS y APIs


En frameworks JS (React, Angular, Vue.js), se cubren eventos, renderizado condicional, formularios, enrutamiento, consumo de APIs y la utilidad de Axios. En React, se destacan props y children para comunicación entre componentes.


DESARROLLO CON FRAMEWORKS JS Y APIS



• DESARROLLAR APLICACIONES CON EVENTOS, CONDICIONALES Y BUCLES EN UN FRAMEWORK JS:
o Eventos:
 Los eventos en el desarrollo de aplicaciones FrontEnd son acciones del usuario o del sistema que desencadenan respuestas específicas en la interfaz. Estos eventos pueden incluir clics del mouse, pulsaciones de teclas, cambios de estado, etc.
 En los frameworks JS como React, Angular y Vue.js, la gestión de eventos se realiza a través de manejadores de eventos. Se asignan funciones específicas a eventos particulares para manejar la lógica asociada.

• RENDERIZADO CONDICIONAL / RENDERIZADO ITERATIVO:
o Condicional: El renderizado condicional permite mostrar u ocultar componentes basándose en ciertas condiciones. Esto es crucial para adaptar la interfaz de usuario a diferentes estados o datos.
o Iterativo: El renderizado iterativo implica la representación de listas o conjuntos de datos mediante bucles. En React, por ejemplo, se puede utilizar el método map para iterar sobre un array y renderizar dinámicamente componentes.

• FORMULARIOS / ROUTING:
o Formularios: Los formularios son elementos cruciales en las aplicaciones web para la recopilación de datos del usuario. En los frameworks JS, la gestión de formularios implica la captura de eventos de entrada y la actualización del estado de la aplicación en consecuencia.
o Routing: El enrutamiento se refiere a la navegación entre diferentes vistas o páginas en una aplicación. Los frameworks proporcionan enrutadores que permiten definir rutas y asociar componentes específicos a esas rutas.

• CONSUMO DE APIS: El consumo de APIs es esencial para obtener y enviar datos desde y hacia un servidor. Las aplicaciones modernas interactúan con APIs para acceder y manipular información. Frameworks como React, Angular y Vue.js ofrecen métodos y herramientas para realizar solicitudes HTTP, ya sea mediante el uso de funciones nativas como fetch o mediante bibliotecas externas como Axios.

• PROMESAS / ASYNC AWAIT:
o Las operaciones asíncronas son comunes en el desarrollo de aplicaciones web, especialmente al interactuar con APIs. Las promesas y la sintaxis async/await simplifican la gestión de este tipo de operaciones.
o Permiten realizar operaciones asíncronas de manera más legible y estructurada, mejorando la mantenibilidad del código.

• OBTENIENDO DATOS DE UNA API: Obtener datos de una API implica realizar solicitudes HTTP. En este contexto, frameworks JS ofrecen métodos para realizar estas solicitudes, ya sea mediante la función fetch o mediante el uso de bibliotecas como Axios. La respuesta de una API se maneja generalmente como un objeto JSON que se puede procesar y utilizar en la aplicación.

• LIBRERÍA AXIOS:
o Definición: Es una librería popular para realizar solicitudes HTTP en JS. Proporciona una interfaz simple y basada en promesas para trabajar con peticiones.
o Ventajas: Permite la configuración de solicitudes, manejo de errores, transformación de datos y funciones que simplifican la interacción con APIs.
o Uso: Para realizar solicitudes HTTP, tanto desde el navegador como desde el entorno de ejecución Node.js.
o Características Principales:
 Facilidad de Uso: Axios proporciona una interfaz simple y fácil de usar para realizar solicitudes HTTP.
 Compatibilidad del Navegador y Node.js: Puede ser utilizada tanto en el navegador como en el entorno de ejecución Node.js, lo que la hace versátil.
 Promesas: Todas las solicitudes en Axios devuelven promesas, lo que facilita el manejo de operaciones asíncronas.
 Manejo Automático de Datos JSON: Axios automáticamente convierte las respuestas JSON en objetos JavaScript, simplificando el proceso de manipulación de datos.
 Configuración de Solicitudes: Permite configurar opciones globales para todas las solicitudes, como encabezados comunes, base URL, y más.
 Interceptores: Axios permite la utilización de interceptores, lo que facilita la manipulación de solicitudes y respuestas antes de que se envíen o después de recibir.

• PROCEDIMIENTOS: En React, para transmitir información de un componente padre a uno hijo, se utilizan props y children. Con props, el componente padre define propiedades que contienen los datos deseados y las pasa al componente hijo durante la renderización. Por otro lado, con children, el componente padre incluye elementos o componentes dentro de la etiqueta del componente hijo. Estos elementos hijos son accesibles en el componente hijo a través de la propiedad especial children. A menudo, se combinan ambos métodos para enviar tanto datos como elementos hijos al componente hijo, facilitando una comunicación efectiva y la construcción de interfaces dinámicas en aplicaciones React.

• RESULTADOS: Se obtuvo experiencia en la gestión de eventos, condicionales y bucles. Se aprendió a adaptar la interfaz con renderizado condicional, manejar formularios, implementar enrutamiento y consumir apis, destacando el uso de Axios para solicitudes HTTP eficientes. Además, se adquirieron habilidades en el manejo de operaciones asíncronas mediante promesas y async/await.
