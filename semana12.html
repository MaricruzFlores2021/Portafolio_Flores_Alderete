Tema de la Semana 12 :
Desarrollo backend y aplicaciones con Spring Core y Spring Boot


En esta semana se proporciona una visión detallada del panorama del desarrollo backend, desde la arquitectura hasta las herramientas y frameworks específicos, destacando el enfoque en Java y Spring para construir aplicaciones eficientes y escalables.


DESARROLLO BACKEND Y APLICACIONES CON SPRING CORE Y SPRING BOOT



• Desarrollo Backend:

o Arquitectura de Aplicaciones Web: La arquitectura de aplicaciones web es esencial para el diseño y la organización eficiente de sistemas. Implica la definición de la estructura modular, la interacción entre los componentes y la implementación de la lógica de negocio. La elección de patrones arquitectónicos como MVC (Modelo-Vista-Controlador) ayuda a crear aplicaciones escalables y mantenibles, dividiendo la lógica en capas bien definidas.
o Servidores Web: Los servidores web son componentes clave en el backend, actuando como intermediarios entre los usuarios y las aplicaciones. Gestionan solicitudes HTTP, proporcionan recursos estáticos y aplican medidas de seguridad. Ejemplos como Apache y Nginx son fundamentales para garantizar una entrega eficiente y segura de contenido.
o Funcionamiento del Server Side: En el desarrollo backend, el server side es donde se ejecuta la lógica de la aplicación. Permite realizar operaciones complejas, interactuar con bases de datos y generar dinámicamente el contenido que se enviará al cliente. Este enfoque facilita el procesamiento de datos y la implementación de la lógica de negocio en el lado del servidor.
o Lenguaje y Framework Backend: Java y Jakarta EE se destacan como opciones populares para el desarrollo backend. Java ofrece una plataforma robusta y orientada a objetos, mientras que frameworks como Spring simplifican el desarrollo proporcionando abstracciones útiles, como la inversión de control y la gestión de dependencias, que mejoran la modularidad y la eficiencia del código.
o Herramientas: IntelliJ Idea es un IDE ampliamente utilizado en el desarrollo Java, ofreciendo funciones avanzadas para la codificación, depuración y gestión de proyectos. Tomcat, como servidor web y contenedor servlet, facilita la ejecución de aplicaciones Java, proporcionando un entorno de desarrollo y prueba eficiente.

• Desarrollar Aplicaciones con Spring Core y Spring Boot:

o Desarrollo con Spring Core: Spring Core proporciona conceptos fundamentales como inversión de control (IoC) e inyección de dependencias. Estos elementos descentralizan la gestión de componentes y mejoran la modularidad del código.
o Desarrollo con Spring Boot: Spring Boot, una extensión de Spring, simplifica la configuración y el desarrollo al proporcionar configuraciones predeterminadas y convenciones, permitiendo una implementación rápida y eficiente de aplicaciones Java.
o Terminología y Características Spring:
 Control: La inversión de control (IoC) es un paradigma donde el contenedor de Spring gestiona la creación y el ciclo de vida de los objetos, reduciendo la dependencia de las clases.
 Dependencias: La inyección de dependencias facilita la conexión de componentes, permitiendo una mayor flexibilidad y mantenibilidad del código.
o Módulos Spring: Spring se organiza en módulos como Spring MVC para el desarrollo de aplicaciones web y Spring Data para facilitar el acceso y la gestión de datos de manera eficiente.
o Servlets y JSP (JavaServer Pages):
 Servlets: Son programas Java que extienden la capacidad de los servidores web para manejar solicitudes HTTP, permitiendo la creación dinámica de contenido.
 JSP: Simplifica la creación de páginas web dinámicas al combinar Java y HTML de manera efectiva.
o Spring Boot: Spring Boot, al simplificar la configuración y el desarrollo, permite la creación rápida de aplicaciones Java. Facilita la implementación de aplicaciones autocontenidas con dependencias mínimas, lo que mejora la eficiencia del desarrollo.

• PROCEDIMIENTOS: El procedimiento detalla la instalación de OpenJDK 21 para Java, IntelliJ Idea como el entorno de desarrollo integrado y Spring Boot para simplificar la creación de aplicaciones. Se hizo pasos desde la descompresión de archivos hasta la configuración del Path del sistema. Además, se realizó la ejecución del proyecto Spring Boot mediante el comando .\mvnw spring-boot:run. Este conjunto de instrucciones facilita a los desarrolladores la preparación de su entorno de desarrollo para proyectos Java y Spring.
